# Roadmap - Input Bar

## Known Issues

- **Visual Glitch when reoping InputBar:** If a user launches an application (e.g., the Calculator), the content from the previous session remains visible for about half a second when the bar is reopened.
- - **Proposed Solution:** Clear the window/input state before the bar closes, rather than when it opens. This will make the transition seamless and unnoticeable to the user, and shouldn't require major changes to the existing code logic.
